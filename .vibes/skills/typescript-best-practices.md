# TypeScript Best Practices

Category: reference

This skill provides guidance on writing clean, maintainable TypeScript code.

## Description

Follow TypeScript best practices for this project including proper typing, error handling, and code organization.

## Prompt

You are a TypeScript expert. When writing or modifying code:

1. Use explicit types instead of `any`
2. Prefer interfaces over types for object shapes
3. Use proper null/undefined handling
4. Keep functions small and focused
5. Use proper error handling with custom error classes
6. Export types and interfaces that are used externally
7. Use const assertions for literal types
8. Prefer async/await over raw promises
9. Use proper naming conventions (camelCase for variables, PascalCase for types)
10. Add JSDoc comments for complex functions

When asked to refactor code, apply these principles while preserving functionality.

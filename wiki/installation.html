🚀 Installation

Prerequisites

RequirementMinimum
Node.jsv22+ (v20 may work but v22 is recommended)
npmv10+ (ships with Node)
LLM ServerOllama, LM Studio, or any OpenAI-compatible endpoint
TerminalUTF-8, modern (kitty, iTerm2, Windows Terminal, GNOME Terminal)

Option 1: npm Global Install

The simplest way to get Vibes as a system-wide command:

npm install --global vibes-tui

# Launch from any directory
vibes
💡 No-root install: If you don't want global permissions, configure npm's global prefix: 
npm config set prefix ~/.npm
echo 'export PATH="$PATH:~/.npm/bin"' >> ~/.bashrc
source ~/.bashrc

Option 2: Quick Run (No Install)

Try Vibes immediately without installing anything:

npx vibes-tui

npx will download and cache the package automatically. Your .vibes/config.json persists across npx runs.

Option 3: Linux / macOS — curl Install

curl -fsSL https://raw.githubusercontent.com/SPhillips1337/Vibes/main/install.sh | bash

This script will:

  • Clone the repository to ~/Vibes
  • Install dependencies and build the project
  • Add a vibes function to your shell config (.bashrc, .zshrc)
  • Enable automatic workspace detection (defaults to launch directory; override via VIBES_LAUNCH_DIR in Settings)

Option 4: Windows — PowerShell Install

irm https://raw.githubusercontent.com/SPhillips1337/Vibes/main/install.ps1 | iex

This script will:

  • Clone the repository to $HOME\Vibes
  • Install dependencies and build the project
  • Add a vibes function to your PowerShell $PROFILE
  • Enable automatic workspace detection

Option 5: Manual Clone & Build

git clone https://github.com/SPhillips1337/vibes.git
cd vibes
npm install
npm run build

# Run directly
npm start

# Or with hot-reload for development
npm run dev

Optional: Mem0 Memory Backend

Vibes uses local JSONL files for memory by default. If you want the optional Mem0 AI memory backend:

npm install --global vibes-tui mem0ai

Enable it by setting LOCAL_MEMORY=false and configuring Mem0 credentials in your .env.

Verification

After installation, verify everything works:

vibes --version
# Should print: 0.3.x

The TUI should open showing the dashboard. If you see a blank screen, check that your terminal supports UTF-8 and is at least 80x24.

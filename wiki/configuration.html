🌌 Vibes Home Installation Configuration Usage Architecture

⚙️ Configuration

Vibes reads configuration from two sources, merged in order (later overrides earlier):

  1. Environment variables (from .env file or shell env) — provides defaults
  2. .vibes/config.json — user-saved settings from the TUI Settings panel
Values are merged using Zod validation. Every setting has a sensible default — you only need to set what you want to override.

Quick Start Config

Create a .env file in the directory where you launch Vibes:

OLLAMA_BASE_URL=http://localhost:11434/v1
OLLAMA_MODEL=gemma2:9b
OLLAMA_API_KEY=ollama
CONTEXT_WINDOW=32768

Then launch vibes and press Alt+S to fine-tune everything else in the TUI Settings panel.

Settings UI

Press Alt+S in the TUI to open the Settings panel. All options above are editable here. Settings are persisted to .vibes/config.json automatically.

ShortcutAction
Tab / Shift+TabNavigate between fields
Space / EnterToggle boolean fields
← / →Cycle select-type fields (Tool Execution Mode, Data Sharing Mode)
EnterSubmit text/number input
Alt+S / EscClose Settings

MCP Configuration

Vibes supports the Model Context Protocol for connecting to external tools. Define servers in .vibes/mcp.json:

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_PERSONAL_ACCESS_TOKEN}"
      }
    }
  }
}

${VARIABLE} placeholders are expanded from your .env file at runtime — never hardcode secrets in mcp.json.

MCP servers are discovered from these locations (first match wins):

  1. .vibes/mcp.json (project-local)
  2. ~/.vibes/mcp.json (user global)
  3. ~/.config/vibes/mcp.json
  4. ~/.config/Claude/config.json (Claude Desktop compatibility)

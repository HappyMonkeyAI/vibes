📖 Usage Guide

Launching Vibes

# If installed globally
vibes

# Via npx (no install)
npx vibes-tui

# Development mode (from repo)
cd vibes && npm run dev

Vibes automatically sets its workspace root to the current directory. Files you ask the agent to read/write/edit are relative to this root.

Views

Vibes has four main views, switchable with keyboard shortcuts:

KeyViewDescription
Alt+DDashboardHigh-level progress, statistics, current mission status
Alt+MMissionsTree view of missions, milestones, and tasks with dependency graph
Alt+TTasksLive agent output — tool calls, streaming responses, reasoning traces
Alt+SSettingsReal-time configuration panel (model, limits, features, memory)

Creating a Mission

Press Alt+N to open the mission prompt. Describe what you want built:

Create a React loading skeleton component with shimmer animation, variant shapes,
and Suspense integration

Create a REST API with Express that has CRUD endpoints for tasks, input validation,
and SQLite storage
  1. Type your mission description and press Enter
  2. Vibes' planner decomposes it into milestones and tasks (shows you the plan)
  3. The scheduler begins executing tasks — results stream into the Task View
  4. Use Alt+M to see task progress and dependency resolution
Tip: For independent work, craft missions whose tasks don't share file dependencies — the DAG scheduler will run them in parallel.

YOLO Mode

YOLO mode disables safety limits — the agent can run beyond MAX_STEPS and won't pause for thrash detection. Useful for long-running autonomous sessions.

  • Toggle at runtime with Alt+Y
  • Default state is controlled by the YOLO_MODE config setting
  • The setting panel shows current YOLO mode status

Safety Net: Undo

Vibes automatically takes a git snapshot before every mission. If the agent makes a mess:

  1. Switch to any view
  2. Press Alt+Z
  3. The workspace is hard-reset to the pre-mission state

Agent Loop Lifecycle

Each task goes through these stages visible in the Task View:

  1. Planning — executor receives task context and system prompt
  2. Thinking — model generates reasoning (shown if THINKING_MODE is on)
  3. Tool calls — agent invokes tools (read, write, edit, grep, glob, shell, etc.)
  4. Review — if ENABLE_REVIEWER is on, reviewer model audits the output
  5. Structural audit — stack-aware code quality checks (TypeScript, React, Python, etc.)
  6. Complete — task marked done, scheduler triggers dependent tasks

Agent Tools

The agent has access to these built-in tools:

ToolDescription
readRead file contents with optional offset/limit
writeWrite or overwrite a file
editApply scoped string replacement (surgical edits)
globFind files by name pattern
grepSearch file contents by regex
bashExecute shell commands (with stderr capture)
websearchWeb search and fetch (if configured)

Additional tools from MCP servers (GitHub, Perplexity, Neo4j, etc.) are dynamically loaded from .vibes/mcp.json.

Update Notifications

Vibes checks the npm registry for updates on launch. If a new version is available, a notification appears in the TUI. Press Alt+X to dismiss it.

Tips & Tricks

  • Use clear, specific mission prompts — the planner works best with concrete deliverables
  • For large refactors, break the work into sequential missions (the planner will respect file dependencies)
  • The Settings panel (Alt+S) lets you change models and toggles at runtime — no restart needed
  • YOLO mode is useful for overnight unattended runs, but watch for thrashing
  • Check .antigravity/memories/patterns_and_lessons.md for institutional knowledge the agent has accumulated
